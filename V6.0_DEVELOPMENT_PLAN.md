# JunLite CRM V6.0 开发计划（简化版）

## 📋 版本概述

JunLite CRM V6.0 是一个功能增强版本，专注于引入**销售线索管理**功能，将系统从基础的客户关系管理系统升级为具备完整销售线索管理能力的CRM平台。

## 🎯 版本目标

- **核心功能**：引入销售线索管理模块，实现线索的全生命周期管理
- **功能完善**：完善线索获取、分配、跟踪、转化等核心功能
- **用户体验**：优化线索管理界面，提升用户操作体验
- **数据统计**：提供线索相关的数据统计和报表

## 🚀 核心功能规划

### 1. 销售线索管理模块

#### 1.1 线索获取
- **手动创建**：
  - 基本信息：姓名、电话、邮箱、公司、职位
  - 来源信息：来源渠道（网站、电话、展会、推荐等）、来源详情
  - 线索描述：记录线索的详细信息和需求
  - 优先级：高、中、低

- **批量导入**：
  - Excel导入：支持Excel文件批量导入线索
  - CSV导入：支持CSV文件批量导入线索
  - 数据校验：导入时进行数据校验
  - 导入日志：记录导入结果和错误

- **API接口**：
  - POST /leads - 创建线索
  - POST /leads/import - 批量导入线索
  - GET /leads - 获取线索列表
  - GET /leads/:id - 获取线索详情
  - PUT /leads/:id - 更新线索
  - DELETE /leads/:id - 删除线索

#### 1.2 线索分配
- **手动分配**：
  - 分配给销售人员：选择销售人员分配线索
  - 批量分配：选择多个线索批量分配
  - 分配记录：记录分配时间和分配人
  - 分配历史：查看线索的分配历史

- **自动分配**：
  - 轮询分配：按顺序轮流分配给销售人员
  - 负载均衡：根据销售人员的工作量均衡分配
  - 地区分配：根据线索地区分配给对应销售人员
  - 规则分配：根据自定义规则自动分配

- **分配管理**：
  - 分配规则：配置自动分配规则
  - 销售人员管理：管理可分配的销售人员
  - 工作量统计：统计销售人员的工作量
  - 重新分配：支持重新分配线索

#### 1.3 线索跟踪
- **状态管理**：
  - 新线索：刚创建的线索
  - 已分配：已分配给销售人员
  - 联系中：销售人员已联系
  - 意向明确：客户有明确购买意向
  - 已转化：转化为正式客户
  - 已关闭：无法转化，关闭线索

- **跟进记录**：
  - 联系方式：电话、邮件、短信、面谈
  - 联系时间：记录联系时间
  - 联系内容：记录沟通内容
  - 联系结果：记录联系结果（成功、失败、待跟进）
  - 下次跟进：设置下次跟进时间
  - 附件：上传沟通附件

- **线索评分**：
  - 自动评分：根据线索行为自动评分
  - 手动评分：销售人员手动调整评分
  - 评分规则：自定义评分规则
  - 评分历史：记录评分变化历史

- **提醒功能**：
  - 跟进提醒：提醒销售人员跟进线索
  - 超时提醒：提醒长时间未跟进的线索
  - 到期提醒：提醒即将到期的线索
  - 提醒方式：系统内提醒、邮件提醒

#### 1.4 线索转化
- **转化流程**：
  - 线索验证：验证线索的真实性和有效性
  - 需求确认：确认客户的具体需求
  - 方案提供：提供产品或服务方案
  - 合同签订：签订销售合同
  - 转化客户：将线索转化为正式客户

- **转化记录**：
  - 转化时间：记录转化时间
  - 转化原因：记录转化原因
  - 转化金额：记录转化金额
  - 转化产品：记录转化的产品
  - 转化人员：记录转化的销售人员

- **转化分析**：
  - 转化率：统计线索转化率
  - 转化周期：分析线索转化周期
  - 转化漏斗：展示线索转化漏斗
  - 转化原因：统计转化和未转化的原因

### 2. 线索报表

#### 2.1 线索统计
- **线索数量**：
  - 总线索数：统计总线索数量
  - 新增线索：统计新增线索数量
  - 已分配线索：统计已分配线索数量
  - 已转化线索：统计已转化线索数量
  - 已关闭线索：统计已关闭线索数量

- **线索趋势**：
  - 日趋势：按日统计线索数量趋势
  - 周趋势：按周统计线索数量趋势
  - 月趋势：按月统计线索数量趋势
  - 年趋势：按年统计线索数量趋势

#### 2.2 线索分析
- **来源分析**：
  - 来源分布：统计各来源的线索数量
  - 来源转化率：统计各来源的转化率
  - 来源效果：分析各来源的效果

- **销售人员分析**：
  - 销售人员线索数：统计各销售人员的线索数量
  - 销售人员转化率：统计各销售人员的转化率
  - 销售人员排行：销售人员业绩排行

- **转化分析**：
  - 转化漏斗：展示线索转化漏斗
  - 转化周期：分析线索转化周期
  - 转化原因：统计转化和未转化的原因

## 🗄️ 数据库设计

### 销售线索表（leads）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键ID |
| name | VARCHAR(100) | 线索姓名 |
| phone | VARCHAR(20) | 电话号码 |
| email | VARCHAR(100) | 邮箱地址 |
| company | VARCHAR(200) | 公司名称 |
| position | VARCHAR(100) | 职位 |
| source | VARCHAR(50) | 来源渠道 |
| source_detail | VARCHAR(200) | 来源详情 |
| description | TEXT | 线索描述 |
| priority | VARCHAR(20) | 优先级（高、中、低） |
| status | VARCHAR(20) | 状态（新线索、已分配、联系中、意向明确、已转化、已关闭） |
| assigned_to | INT | 分配给的销售人员ID（关联users表） |
| assigned_at | DATETIME | 分配时间 |
| score | INT | 线索评分 |
| converted | BOOLEAN | 是否已转化 |
| converted_at | DATETIME | 转化时间 |
| converted_amount | DECIMAL(10,2) | 转化金额 |
| closed | BOOLEAN | 是否已关闭 |
| closed_at | DATETIME | 关闭时间 |
| close_reason | VARCHAR(200) | 关闭原因 |
| created_by | INT | 创建人ID（关联users表） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 线索跟进记录表（lead_followups）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键ID |
| lead_id | INT | 线索ID（关联leads表） |
| contact_method | VARCHAR(20) | 联系方式（电话、邮件、短信、面谈） |
| contact_time | DATETIME | 联系时间 |
| contact_content | TEXT | 联系内容 |
| contact_result | VARCHAR(20) | 联系结果（成功、失败、待跟进） |
| next_followup | DATETIME | 下次跟进时间 |
| attachment | VARCHAR(500) | 附件路径 |
| created_by | INT | 创建人ID（关联users表） |
| created_at | DATETIME | 创建时间 |

### 线索分配历史表（lead_assignments）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键ID |
| lead_id | INT | 线索ID（关联leads表） |
| assigned_to | INT | 分配给的销售人员ID（关联users表） |
| assigned_by | INT | 分配人ID（关联users表） |
| assigned_at | DATETIME | 分配时间 |
| notes | VARCHAR(500) | 分配备注 |
