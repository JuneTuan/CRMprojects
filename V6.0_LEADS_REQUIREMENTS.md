## 1. 项目背景

### 1.1 背景
JunLite CRM V5.2R1版本已经具备了基础的客户管理、产品管理、订单管理等功能，但缺乏销售线索管理能力。在实际销售过程中，企业需要从多个渠道获取销售线索，并对线索进行有效的管理和跟踪，以提高销售效率和转化率。

### 1.2 目标
引入销售线索管理功能，实现线索的全生命周期管理，包括线索获取、分配、跟踪、转化等环节，帮助企业提高销售效率和转化率。

### 1.3 范围
本需求文档涵盖销售线索管理的以下功能：
- 线索获取（手动创建、批量导入）
- 线索分配（手动分配、自动分配）
- 线索跟踪（状态管理、跟进记录、线索评分）
- 线索转化（转化流程、转化分析）
- 线索报表（线索统计、线索分析）

## 2. 功能需求

### 2.1 线索获取

#### 2.1.1 手动创建线索
**需求描述**：销售人员可以手动创建销售线索。

**功能要求**：
1. 录入线索基本信息
   - 姓名（必填，最多100字符）
   - 电话号码（必填，最多20字符）
   - 邮箱地址（选填，最多100字符）
   - 公司名称（选填，最多200字符）
   - 职位（选填，最多100字符）

2. 录入来源信息
   - 来源渠道（必填，下拉选择：网站、电话、展会、推荐、其他）
   - 来源详情（选填，最多200字符）

3. 录入线索描述
   - 线索描述（选填，文本框，最多2000字符）
   - 记录线索的详细信息和需求

4. 设置优先级
   - 优先级（必填，下拉选择：高、中、低）
   - 高优先级的线索应该优先处理

5. 分配销售人员
   - 可以在创建时直接分配给销售人员
   - 也可以创建后再分配

**验收标准**：
- 可以成功创建线索
- 必填项有验证提示
- 创建成功后跳转到线索列表
- 创建失败时显示错误信息

#### 2.1.2 批量导入线索
**需求描述**：支持通过Excel或CSV文件批量导入线索。

**功能要求**：
1. 支持Excel文件导入
   - 文件格式：.xlsx
   - 文件大小：不超过10MB
   - 字段映射：自动识别或手动映射字段

2. 支持CSV文件导入
   - 文件格式：.csv
   - 文件大小：不超过10MB
   - 编码格式：UTF-8

3. 数据校验
   - 必填项校验：姓名、电话、来源必填
   - 数据格式校验：电话格式、邮箱格式
   - 重复数据检查：检查是否已存在相同线索

4. 导入日志
   - 显示导入进度
   - 显示成功导入数量
   - 显示失败数量和原因
   - 生成导入报告

**验收标准**：
- 可以成功导入Excel和CSV文件
- 数据校验准确
- 导入日志清晰
- 导入失败时有明确的错误提示

### 2.2 线索分配

#### 2.2.1 手动分配线索
**需求描述**：管理员可以手动将线索分配给销售人员。

**功能要求**：
1. 单个线索分配
   - 在线索详情页面可以分配给销售人员
   - 选择销售人员后保存分配信息
   - 记录分配时间和分配人

2. 批量线索分配
   - 在线索列表页面可以批量选择线索
   - 批量分配给同一个销售人员
   - 记录每条线索的分配信息

3. 分配历史
   - 可以查看线索的分配历史
   - 显示每次分配的时间、分配人、被分配人
   - 支持按时间排序

**验收标准**：
- 可以成功分配线索
- 分配记录完整
- 分配历史可查询

#### 2.2.2 自动分配线索
**需求描述**：系统可以根据预设规则自动分配线索给销售人员。

**功能要求**：
1. 轮询分配
   - 按顺序轮流分配给销售人员
   - 确保每个销售人员获得大致相等的线索数量
   - 可以设置轮询的起始销售人员

2. 负载均衡分配
   - 根据销售人员当前的工作量分配
   - 优先分配给工作量少的销售人员
   - 工作量包括：未处理的线索数量、跟进中的线索数量

3. 地区分配
   - 根据线索的所在地区分配
   - 将线索分配给对应地区的销售人员
   - 需要维护销售人员的负责地区

4. 规则分配
   - 支持自定义分配规则
   - 可以设置多个条件：来源、优先级、地区等
   - 满足条件的线索自动分配给指定销售人员

**验收标准**：
- 自动分配规则可以配置
- 自动分配准确执行
- 分配结果可查看

#### 2.2.3 分配管理
**需求描述**：管理自动分配规则和销售人员信息。

**功能要求**：
1. 分配规则配置
   - 创建分配规则
   - 编辑分配规则
   - 删除分配规则
   - 启用/禁用分配规则

2. 销售人员管理
   - 查看销售人员列表
   - 设置销售人员是否可分配线索
   - 设置销售人员负责的地区
   - 查看销售人员的工作量

3. 工作量统计
   - 统计销售人员当前的工作量
   - 包括：未处理线索数、跟进中线索数、已转化线索数
   - 支持按时间范围统计

4. 重新分配
   - 支持将线索重新分配给其他销售人员
   - 记录重新分配的原因
   - 保留原分配记录

**验收标准**：
- 分配规则可以灵活配置
- 销售人员信息管理完整
- 工作量统计准确
- 重新分配功能可用

### 2.3 线索跟踪

#### 2.3.1 状态管理
**需求描述**：线索有明确的状态流转，销售人员可以更新线索状态。

**功能要求**：
1. 线索状态定义
   - 新线索：刚创建的线索
   - 已分配：已分配给销售人员
   - 联系中：销售人员已联系
   - 意向明确：客户有明确购买意向
   - 已转化：转化为正式客户
   - 已关闭：无法转化，关闭线索

2. 状态流转规则
   - 新线索 → 已分配：分配给销售人员后
   - 已分配 → 联系中：销售人员开始联系
   - 联系中 → 意向明确：客户有明确意向
   - 意向明确 → 已转化：签订合同
   - 任意状态 → 已关闭：无法转化时关闭

3. 状态更新
   - 销售人员可以更新线索状态
   - 更新时需要填写状态变更原因
   - 记录状态变更历史

**验收标准**：
- 状态定义清晰
- 状态流转规则合理
- 状态更新有记录

#### 2.3.2 跟进记录
**需求描述**：销售人员可以记录线索的跟进情况。

**功能要求**：
1. 添加跟进记录
   - 选择联系方式：电话、邮件、短信、面谈
   - 填写联系时间：默认当前时间，可修改
   - 填写联系内容：记录沟通的详细内容
   - 选择联系结果：成功、失败、待跟进
   - 设置下次跟进时间：可选
   - 上传附件：可选

2. 查看跟进记录
   - 在线索详情页面显示跟进记录列表
   - 按时间倒序排列
   - 显示每次跟进的详细信息
   - 支持下载附件

3. 跟进提醒
   - 根据下次跟进时间提醒销售人员
   - 提醒方式：系统内提醒、邮件提醒
   - 提醒时间：提前1天、当天、超时

**验收标准**：
- 可以成功添加跟进记录
- 跟进记录显示完整
- 跟进提醒准确

#### 2.3.3 线索评分
**需求描述**：系统可以对线索进行评分，帮助销售人员识别高价值线索。

**功能要求**：
1. 自动评分
   - 根据线索信息自动评分
   - 评分因素：来源、优先级、公司规模、职位等
   - 评分规则可以配置

2. 手动评分
   - 销售人员可以手动调整线索评分
   - 评分范围：0-100分
   - 记录评分原因

3. 评分历史
   - 记录线索的评分变化历史
   - 显示每次评分的时间、评分人、评分值
   - 支持查看评分趋势

**验收标准**：
- 自动评分规则可配置
- 手动评分功能可用
- 评分历史完整

### 2.4 线索转化

#### 2.4.1 转化流程
**需求描述**：销售人员可以将线索转化为正式客户。

**功能要求**：
1. 线索验证
   - 验证线索的真实性和有效性
   - 记录验证结果
   - 无法验证的线索可以关闭

2. 需求确认
   - 确认客户的具体需求
   - 记录客户需求详情
   - 评估需求匹配度

3. 方案提供
   - 提供产品或服务方案
   - 记录方案内容
   - 上传方案文档

4. 合同签订
   - 记录合同签订信息
   - 上传合同文档
   - 记录合同金额

5. 转化客户
   - 将线索转化为正式客户
   - 自动创建客户档案
   - 关联线索和客户

**验收标准**：
- 转化流程完整
- 转化信息记录完整
- 转化后自动创建客户

#### 2.4.2 转化分析
**需求描述**：系统提供线索转化分析功能。

**功能要求**：
1. 转化率统计
   - 计算线索转化率
   - 按时间范围统计：日、周、月、年
   - 按销售人员统计
   - 按来源统计

2. 转化周期分析
   - 分析线索从创建到转化的周期
   - 计算平均转化周期
   - 识别转化周期过长或过短的线索

3. 转化漏斗
   - 展示线索转化漏斗
   - 漏斗阶段：新线索 → 已分配 → 联系中 → 意向明确 → 已转化
   - 显示每个阶段的数量和转化率

4. 转化原因
   - 统计转化的原因
   - 统计未转化的原因
   - 分析转化和未转化的关键因素

**验收标准**：
- 转化率统计准确
- 转化周期分析合理
- 转化漏斗清晰
- 转化原因分析全面

### 2.5 线索报表

#### 2.5.1 线索统计
**需求描述**：提供线索的统计数据。

**功能要求**：
1. 线索数量统计
   - 总线索数：统计总线索数量
   - 新增线索：统计新增线索数量
   - 已分配线索：统计已分配线索数量
   - 已转化线索：统计已转化线索数量
   - 已关闭线索：统计已关闭线索数量

2. 线索趋势
   - 日趋势：按日统计线索数量趋势
   - 周趋势：按周统计线索数量趋势
   - 月趋势：按月统计线索数量趋势
   - 年趋势：按年统计线索数量趋势
   - 支持图表展示：折线图、柱状图

**验收标准**：
- 线索数量统计准确
- 线索趋势图表清晰
- 支持多种时间维度

#### 2.5.2 线索分析
**需求描述**：提供线索的分析功能。

**功能要求**：
1. 来源分析
   - 来源分布：统计各来源的线索数量
   - 来源转化率：统计各来源的转化率
   - 来源效果：分析各来源的效果
   - 支持饼图、柱状图展示

2. 销售人员分析
   - 销售人员线索数：统计各销售人员的线索数量
   - 销售人员转化率：统计各销售人员的转化率
   - 销售人员排行：销售人员业绩排行
   - 支持表格、图表展示

3. 转化分析
   - 转化漏斗：展示线索转化漏斗
   - 转化周期：分析线索转化周期
   - 转化原因：统计转化和未转化的原因
   - 支持漏斗图、饼图展示

**验收标准**：
- 来源分析全面
- 销售人员分析准确
- 转化分析深入

## 5. 业务流程

### 5.1 线索获取流程

```
开始
  ↓
选择创建方式（手动/导入）
  ↓
手动创建 → 填写线索信息 → 保存线索
  ↓
批量导入 → 选择文件 → 上传文件 → 数据校验 → 导入线索
  ↓
结束
```

### 5.2 线索分配流程

```
开始
  ↓
选择分配方式（手动/自动）
  ↓
手动分配 → 选择线索 → 选择销售人员 → 确认分配
  ↓
自动分配 → 配置分配规则 → 系统自动分配
  ↓
记录分配信息
  ↓
结束
```

### 5.3 线索跟踪流程

```
开始
  ↓
查看线索详情
  ↓
添加跟进记录
  ↓
更新线索状态
  ↓
设置下次跟进时间
  ↓
系统提醒跟进
  ↓
结束
```

### 5.4 线索转化流程

```
开始
  ↓
验证线索
  ↓
确认需求
  ↓
提供方案
  ↓
签订合同
  ↓
转化为客户
  ↓
结束
```
